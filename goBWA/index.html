Important concepts in Power Automate

  • Every flow has two main parts: a trigger, and one or more actions.
  • You can think of the trigger as the starting action for the flow. The trigger can be something like a new email arriving in your inbox or a new item being added to a SharePoint list.
  • Actions are what you want to happen when a trigger is invoked. For example, the new email trigger will start the action of creating a new file on OneDrive for Business. Other examples of actions include sending an email, posting a tweet, and starting an approval.

Get claat

Download the binary

  1. For macOS (assuming binary is claat-darwin-amd64), set executable permissions and copy the binary to a location specified in the $PATH environment variable
chmod 555 claat-darwin-amd64
cp claat-darwin-amd64 /usr/local/bin/
  1. For other platforms, copy the binary to a location specified in the $PATH environment variable

claat export (create static html pages)

  1. Obtain the google doc id from the URL of the Google Doc (right after docs.google.com/document/d/)
  2. Run the command shown below
  • use the platform-specific claat executable name (e.g. claat-darwin-amd64 for macOS, claat-windows-386.exe for Windows)
  • replace googledocid with the google doc id
  • a folder based on the URL field of the codelab metadata will be created
claat-darwin-amd64 export googledocid

claat serve (download dependencies and serve content locally)

  1. Run the command shown below
  • two folders will be created - elements and bower_components
  • the codelab will be served locally
claat-darwin-amd64 serve

The claat serve command must be run at least once to download the dependencies, even if there is no intention to serve the content locally.

Set up Google App Engine

  1. Go to Google App Engine and create an account
  2. Go to Google App Engine console and create a new project

Host static content on Google App Engine (Reference)

  1. Download, install and initalize the Google Cloud SDK
  • add the bin folder to the $PATH environment variable
  1. Create a content folder that has the same name as your project ID. You can find your project ID in the Console.
  2. In the content folder, create a file app.yaml

app.yaml

runtime: python27
api_version: 1
threadsafe: true

handlers:
- url: /
  static_files: www/index.html
  upload: www/index.html

- url: /(.*)
  static_files: www/\1
  upload: www/(.*)
  1. In the content folder, create a folder www
  2. From the www folder
  • run the claat export command (see previous step)
  • run the claat serve command (see previous step)

Once the elements and bower_components folders are created in the www folder, you can use the claat export command below from the content folder and skip the claat serve command:

claat-darwin-amd64 export -o www googledocid
  1. From the content folder
  • run the gloud app deploy command
gcloud app deploy -v v1

On first run, specify the geographic location to deploy to. For Singapore, use asia-east2.

The -v switch is used to specify the version of the app. App engine allows up to 15 versions (free) / 120 versions (paid) of an app before previous versions have to be deleted.

  • run the gloud app browse command (optional)
gcloud app browse

The URL of your Codelab will be https://appengineappid.appspot.com/urlname/index.html (where appengineappid is the app engine project ID and urlname is the URL field from the codelab metadata)
